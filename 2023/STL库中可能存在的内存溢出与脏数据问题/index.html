  STL 库中的 vector 是我们使用最频繁的 STD 容器之一。它具有广泛的应用，并且在性能方面表现出色。然而，其存在一种潜在问题，即 溢出。由于 vector 在使用下标访问元素时不会检查索引是否越界，因此很可能导致溢出错误的出现。这种错误被严格定义为一种内存的"dirty read / dirty write"。
